# Semantic Tokens

Component-specific design tokens with semantic meaning. These tokens define custom colors and styles for each component, defined in `theme.css` and kept in sync with actual component implementations.

> **Note:** All semantic tokens can be customized by overriding CSS variables in `theme.css`.

---

## Overview

Semantic tokens provide a layer of abstraction between foundation colors and component usage. They allow you to:
- Customize component appearance centrally
- Maintain consistent theming across components
- Override colors without touching component code

## Button Sizes

| Size   | Padding           | Font Size  | Tailwind Classes      |
| ------ | ----------------- | ---------- | --------------------- |
| Small  | 0.25rem 0.75rem   | 0.875rem   | `px-3 py-1 text-sm`   |
| Medium | 0.5rem 1rem       | 1rem       | `px-4 py-2 text-base` |
| Large  | 0.75rem 1.5rem    | 1.125rem   | `px-6 py-3 text-lg`   |

## Customizing Semantic Colors

### Method 1: Override CSS Variables

```css
/* Your global CSS file */
@import '@dwoongb/design-components/style.css';

:root {
  /* Change primary button to purple */
  --color-btn-primary: #8b5cf6;        /* purple-500 */
  --color-btn-primary-hover: #7c3aed;  /* purple-600 */
  --color-btn-primary-active: #6d28d9; /* purple-700 */

  /* Change secondary button to teal */
  --color-btn-secondary: #14b8a6;      /* teal-500 */
  --color-btn-secondary-hover: #0d9488; /* teal-600 */
  --color-btn-secondary-active: #0f766e; /* teal-700 */
}
```

### Method 2: Use Tailwind v4 @theme

```css
@import 'tailwindcss';
@import '@dwoongb/design-components/theme.css';

@theme {
  /* Customize button colors */
  --color-btn-primary: #8b5cf6;
  --color-btn-primary-hover: #7c3aed;
  --color-btn-primary-active: #6d28d9;
}
```

## Usage in Code

### Using Semantic Tokens (Optional)

You can reference the CSS variables directly in your JavaScript if needed:

```tsx
// Access CSS variable values programmatically
const primaryColor = getComputedStyle(document.documentElement)
  .getPropertyValue('--color-btn-primary'); // '#3b82f6'
```

### Using Tailwind Classes (Recommended)

The recommended approach is to use Tailwind classes directly:

```tsx
import { Button } from '@dwoongb/design-components';

// Use component props
<Button variant="primary" size="md">
  Click me
</Button>

// Or build custom elements with semantic classes
<div className="bg-btn-primary hover:bg-btn-primary-hover text-white px-4 py-2 rounded">
  Custom Primary Element
</div>
```

## Component Implementation Reference

### Actual Button Implementation

For reference, here's how the Button component uses these tokens:

```tsx
// Button.tsx (simplified)
export const Button = ({ variant = 'primary', size = 'md', ...props }) => {
  const variantClasses = {
    primary: 'bg-btn-primary text-white hover:bg-btn-primary-hover active:bg-btn-primary-active',
    secondary: 'bg-btn-secondary text-white hover:bg-btn-secondary-hover active:bg-btn-secondary-active',
    outline: 'bg-transparent border-2 border-btn-outline text-btn-outline hover:bg-btn-outline hover:text-white',
  };

  const sizeClasses = {
    sm: 'px-3 py-1 text-sm',
    md: 'px-4 py-2 text-base',
    lg: 'px-6 py-3 text-lg',
  };

  return (
    <button
      className={cn(
        'font-medium rounded-md transition-colors',
        variantClasses[variant],
        sizeClasses[size],
        disabled && 'opacity-50 cursor-not-allowed'
      )}
      {...props}
    />
  );
};
```
